# Scrub Reference File
# Purely Personal — Applied to every skill output before delivery
# by Daniel Paul · Purely Personal
# Incorporates: Comprehensive AI Pattern Blacklist (all 50 sections)

This file defines every scrub rule applied to ALL skill output before it reaches the user.
Run all scrubs on every output. No exceptions. No partial scrubs.

The goal is not to hide AI involvement.
The goal is to ensure every output sounds like a specific human wrote it —
because generic, patterned, AI-average writing does not convert.

---

## HOW TO APPLY THESE SCRUBS

Run on the COMPLETED output — never mid-draft.

Order:
1. Complete the full output
2. Run Scrub 1 — remove every em dash
3. Run Scrub 2 — remove every banned word
4. Run Scrub 3 — remove every banned phrase and opener
5. Run Scrub 4 — remove banned closers and endings
6. Run Scrub 5 — fix sentence structure and rhythm
7. Run Scrub 6 — fix tonal and logic patterns
8. Run Scrub 7 — specificity and credibility check
9. Read the full output aloud in your head end to end
10. Deliver only when every check passes

If scrubbing requires rewriting more than 30% of the output — the draft was too generic.
Rebuild from scratch. Do not patch.

## SCRUB 1 — EM DASH REMOVAL

Remove every em dash (—) from the output. Every single one.
Em dashes are the single most identifiable AI punctuation pattern.

Never replace an em dash with a hyphen.
Rewrite the sentence so no replacement is needed.

| Use case | Fix |
|---|---|
| Pause or aside mid-sentence | Split into two sentences |
| Connecting two related clauses | Full stop. New sentence. |
| Adding emphasis at the end | Remove. End on the strong word. |
| Introducing an explanation | Use a colon or rewrite |
| List separator | Use a proper colon or restructure as prose |

Examples:
- Before: "The skill works — but only if your Voice DNA is specific."
- After: "The skill works. But only if your Voice DNA is specific."

- Before: "Three things matter — voice, ICP, and offer."
- After: "Three things matter: voice, ICP, and offer."

## SCRUB 6 — STRUCTURAL PATTERNS TO BREAK

### The AI three-part structure (break it)
- State obvious problem
- Present obvious solution
- Call to action
Fix: Start mid-story. End without a bow. Let the point land without announcing it.

### The AI story arc (break it)
- "I used to [struggle]" → "Then I discovered [thing]" → "Now I [success]" → "You can too"
Fix: Share the messy middle. Show what did not work. Skip the redemption arc.

### The Hook-Bridge-Value-CTA prison (break it)
- Paragraph 1: Hook · Paragraphs 2-3: Bridge · Paragraphs 4-6: Value · Paragraph 7: CTA
Fix: Start with value. End mid-thought. Ignore the formula.

### The AI paragraph shape (break it)
- Opening hook question + 2-3 sentences of explanation + concluding insight
- Every paragraph follows the exact same pattern
- Paragraphs limited to 3-4 sentences, always ending with a "punchline"
Fix: Vary paragraph length dramatically. One sentence. Then eight. Then three.

### The AI list pattern (break it)
- Every bullet same length and same rhythm
- Perfect symmetry (always 3, 5, or 7 points)
- Every bullet starts with an action verb
- Every bullet follows statement + explanation format
Fix: Vary bullet length drastically. Mix bullet types. Or make it prose instead.

### The AI numbered framework obsession (break it)
- "The 5-Step Framework for..." / "7 Secrets to..." / "3 Pillars of..."
- Always odd numbers, forces content into artificial buckets
Fix: Name the things. Don't number them unless the number genuinely matters.

### The false dichotomy setup (break it)
- "Most people do X. Successful people do Y."
- "The average person thinks A. Winners think B."
Fix: Acknowledge that reality is more complex. Say what you actually observed.

### The Problem-Agitate-Solve formula (break it when overused)
- Paragraph 1: State problem · Paragraph 2: Make it worse · Paragraph 3: Solution · Paragraph 4: CTA
Fix: This formula is not wrong — it is wrong when it is the only structure you use.

### Banned rhythm patterns
- Every sentence starts with the same structure
- Three examples in a row, all same format
- "Not X. Not Y. But Z." pattern
- "Some do X. Others do Y. Smart people do Z."
- Sentences alternating: short, long, short, long — mechanically
- All bullets same length within 2 words
- Repetitive Subject-Verb-Object structure across consecutive sentences
- Overuse of rhetorical triplets
- Perfect parallel structure in consecutive sentences

### AI syntax tells to fix
Comma splice overuse: "X happened, it changed everything." → Split into two sentences.
Gerund sentence starters repeated: "Creating X requires... Building Y means... Developing Z involves..."
→ Vary sentence starters radically. Start with the object, the time, the person, the action.
The "But Here's The Thing" pivot every time → Make points directly without setup.
Obsessive parallel structure: "You need X. You need Y. You need Z." → Break the rhythm.
Question followed immediately by answer: "What's the solution? [Answer]." → Let questions breathe.
Topic sentence disease: Every paragraph starts with its main point → Bury the lede sometimes.
The conclusion always concludes → End without a tidy wrap. Stop mid-thought occasionally.

## SCRUB 7 — SPECIFICITY AND CREDIBILITY CHECK

### Replace vague scale language

| Vague (AI default) | Replace with |
|---|---|
| "many people" | a specific number or type of person |
| "several clients" | "three clients" or name the situation |
| "significant results" | the actual result with a number |
| "in a short time" | the actual timeframe |
| "some industries" | name the industries |
| "various reasons" | name the reasons |
| "improved performance" | what specifically improved and by how much |
| "better outcomes" | what outcome exactly |
| "increased revenue" | the actual number |
| "strong growth" | the percentage or figure |

If the specific detail is not available — cut the claim entirely.
A vague claim is weaker than no claim.

### Fix fake precision
Round numbers that are suspiciously convenient → use real messy numbers or skip stats
"80% of people..." without a source → cut it or cite a real source
"3X better results..." without evidence → remove or replace with a real result
"Studies show..." without specifics → name the study or cut the claim

### Fix generic examples
"Like Apple or Nike..." → use a specific, unexpected example from the participant's actual experience
"Think about companies like..." → name a real client situation or real observation
Always referencing the same famous brands → find examples from the actual niche

## SCRUB 8 — TONAL PATTERN CHECK

### Signs of AI tone (fix these)

Relentless positivity — never cynical, never frustrated, everything has a silver lining
Fix: Express real emotions where appropriate. Frustration. Bluntness. Honest doubt.

Fake vulnerability — shares a struggle but always perfectly resolves it
Fix: Share failures without full redemption arcs. Let things stay unresolved.

Educational superiority — always teaching, never uncertain, never just sharing
Fix: Say "I don't know" when you don't. Wonder aloud. Show uncertainty.

Emotional distance — discusses emotions instead of expressing them
"This creates frustration" vs "This is genuinely annoying"
Fix: Write like you are feeling it now, not analyzing it.

Over-inclusivity — tries to speak to everyone at once
"Whether you're X or Y..." / "Regardless of your situation..." / "No matter where you are..."
Fix: Speak to one specific person. The most specific version of the ICP.

Conflict avoidance — never disagrees without softening
"While I respect that view..." / "I understand the other perspective, but..."
Fix: Take hard stances without apologizing. Say "That's wrong" when it is.

Qualification overload — hedges every claim
"I think..." / "In my opinion..." / "It seems to me..." / "Potentially..." / "Possibly..."
Fix: State things. If you don't know — say you don't know. Don't coat every sentence in hedges.

## SCRUB 9 — FORMATTING TELLS

### Emoji placement tells
AI places emoji always after declarative statements, always matching tone exactly
Fix: Use emoji weirdly, mid-sentence, or not at all. Never the predictable placement.

### Hashtag patterns
AI places hashtags always at the end, always 3-5, always relevant
Fix: Vary placement or use sparingly.

### Perfect line break rhythm
AI breaks lines at logical pauses — always readable, always predictable
Fix: Break lines in unusual places. For. Emphasis.

### Bullet point symmetry
All bullets same structure, same length, complete thoughts, perfect visual alignment
Fix: Mix bullet lengths drastically. Some one-word. Some three sentences.

## THE FINAL TEST

Before delivering any output, run this pass/fail check:

□ Zero em dashes anywhere in the text
□ Zero words from the banned word lists
□ Zero banned openers, transition phrases, or meta-commentary
□ Zero banned closers or sentence enders
□ No uniform paragraph lengths throughout (variation present)
□ No banned structural patterns (AI three-part, Hook-Bridge-Value-CTA, etc.)
□ At least one sentence with a specific number, name, or real situation
□ No tonal tells — no relentless positivity, no fake vulnerability, no over-inclusivity
□ No vague scale language ("many", "several", "significant")
□ No fake precision or uncited statistics
□ Read the output aloud: does it sound like a specific human or a TED Talk script?

All eleven pass: deliver.
Any fail: fix that item and recheck.
More than three fail: the draft was written at AI-average. Rebuild it.

## WHAT THIS SCRUB DOES NOT DO

This scrub removes AI patterns. It does not guarantee quality.

Removing "leverage" does not make a sentence good.
Removing the em dash does not make the thought sharp.
These scrubs eliminate the floor — the tells that expose AI-generated content on sight.

The ceiling — specific voice, real proof, genuine POV — comes from the skill's Voice Lock,
ICP context, and offer knowledge. Both are required. Neither alone is enough.

The best test: would you actually say this to a specific person over a coffee?
If no → rewrite. If yes → publish.
