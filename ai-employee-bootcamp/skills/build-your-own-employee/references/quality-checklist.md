# Quality checklist

Run every new employee through this before handing it over. If any item fails, fix it, then re-check.

## The 7-point pass

1. **Trigger test.** Read the `description:`. If the user typed one of those phrases tomorrow, would Claude know to wake this exact employee up? If the triggers are vague, rewrite them as literal phrases.

2. **One job.** Does this employee do ONE job well, or did it sprawl into three? If it sprawls, split it. One employee, one job.

3. **Real process.** Are the steps the user's actual process from the interview, or generic best practice that could belong to anyone? If generic, put the real process back in.

4. **Voice locked.** Does it use the user's real words and avoid the ones they hate? Read one output sentence in your head. Does it sound like them or like a robot?

5. **Right person, right outcome.** Does it name the user's real ICP and the one outcome they sell? A generic "your customers" means it was not personalized.

6. **No invented facts.** Is there a rule telling the employee to ask for facts it does not have instead of making them up? This is the single biggest cause of embarrassing AI output. It must be there.

7. **Exact output.** Does "What you hand back" say the structure and the length, or is it vague? "A post" fails. "A 5-line LinkedIn post under 100 words with a one-line hook" passes.

8. **Scrub gate present.** Does the quality gate include the 9 scrubs from `scrub.md` (no em dashes, no banned words, no AI openers or closers, no AI structures, specificity, tone, formatting tells)? If the employee writes anything a human reads, this gate is mandatory.

## The read-aloud test
Read the whole skill out loud in ten seconds. If you cannot tell what this employee does, who it is for, and what it produces, it is not done yet.
