How to read Terminal commands (if this is new)
Lines inside the dark boxes below are commands. Copy the whole line, paste into your terminal, press Enter. Click the 📋 icon on any command to copy it with one click. You're safe. Every command here has been run by hundreds of people. Typos don't break your computer; just re-run the command.
⚠ Windows users · use Git Bash, not PowerShell or Command Prompt.
Every command in this guide uses Unix-style syntax (~/, $(pwd), ln -s, claude). PowerShell does not understand these and will return "command not recognized". Git Bash was installed alongside Git in Step 3 · find it in your Start menu and use it for every step. If you're seeing red errors in PowerShell, switch to Git Bash and re-run.
Stuck at any step? Post in the workshop Slack channel (link in your registration email). Danny or a teammate will unblock you within 30 minutes during Finland business hours.

Claude says "I don't have Bash access" or refuses to run commands
You're probably in Claude Desktop chat, not Claude Code CLI. Claude Code (the terminal app) has Bash. Claude Desktop chat does not (yet · it has File and Web search, but not raw Bash). Open Claude Code via claude in your terminal and paste the Setup Prompt there.

Connector · Required

Connect Gmail

⏱ 2 min · OAuth · powers 4 routines
+

What Gmail unlocks

Gmail is used by 4 of 6 weekly routines. Without it, half the autopilot stops working:

  • Monday 7am · Content Drop · scans your inbox for audience questions to seed this week's posts
  • Thursday 7am · Meeting Prep · drafts a 3-min brief per external meeting attendee, saves to Drafts
  • Friday 9am · Weekly Wrap · scans replies + engagement signals, sends a self-summary email
  • Sunday 6pm · Pipeline Report · counts outreach sent + replies received this week

How to connect (3 clicks)

  1. Open claude.ai/customize/connectors
  2. Click the + button at the top of the Connectors panel → Browse connectors → find Gmail → click Add
  3. Sign in with the Google account you actually use for the business · then approve every permission below

Permissions to approve (all 4)

✓

Tick every box. Skip "send drafts" and Thursday Meeting Prep silently fails (Claude composes the brief but can't save it). The fastest way: click "Allow all" if Google offers that option.
Back on the Connectors page, Gmail shows a green "● Connected" badge. Test by asking Claude in your business folder: "Read the last 3 emails I received today."
Common Gmail issues
"Routines fail silently · I never see drafts." You skipped "Compose & send drafts" during OAuth. Go to myaccount.google.com/permissions, find Claude, click Remove access, then reconnect from claude.ai/customize/connectors and tick every box.

"My Google Workspace admin blocked third-party OAuth." Ask your IT admin to allow Claude as an approved third-party app. Or use a personal @gmail.com account for the workshop and forward business emails to it.

"I have multiple Gmail accounts · which one should I connect?" The one you treat as your business inbox. The one your prospects + clients email. Routines read this inbox · pick the one with real signal.

Connector · Required

Connect Notion

⏱ 2 min · OAuth · powers 4 routines
+

What Notion unlocks

Notion is the database brain of the system. 4 of 6 routines write here:

  • Monday Content Drop · 3 LinkedIn posts + newsletter idea saved to your Content Calendar database
  • Tuesday Lead Research · 10 prospects scored + saved to your CRM database with BANT scores
  • Wednesday Newsletter Draft · pulls topic from Content Calendar to draft full newsletter
  • Sunday Pipeline Report · queries CRM deals + summarizes status

Setup tip · prep your workspace first

Claude can write to ANY Notion workspace, but the routines work best when you have at minimum two databases ready:

  • A page or database called Content Calendar · stores draft posts
  • A database called CRM with columns: Name, Company, Role, Status, BANT Score, Notes

Don't have these yet? Skip · Day 1 of the workshop walks you through creating them. Or ask Claude Code: "Create my Purely Personal Notion workspace structure."

How to connect (4 clicks)

  1. Connectors page → + → Browse connectors → Notion → Add
  2. Sign in to Notion (use the workspace you treat as your business workspace)
  3. Pick which pages Claude can access. Recommended: select the whole workspace · Claude only writes where you tell it to.
  4. Click Allow access
"Selected pages" vs "Entire workspace". If you pick "Selected pages", Claude can only write to pages you explicitly tick. Easier security, but you'll have to come back here every time you create a new database. Pick "Entire workspace" if you trust Claude to write only where instructed (it does · it always asks before writing).
Back on Connectors, Notion shows "● Connected" with the workspace name. Test: ask Claude "List the top-level pages in my Notion workspace."
Common Notion issues
"Claude can't find my CRM database." The CRM database needs the exact name CRM (case-sensitive). Or tell Claude the actual name: "My CRM is called 'Lead Tracker'."

"I have multiple Notion workspaces · how do I switch?" Disconnect Notion from claude.ai/customize/connectors. Reconnect and pick the other workspace at the OAuth step.

"Routines fail with 'page not found' errors." You connected with "Selected pages" and didn't tick the Content Calendar / CRM. Reconnect with "Entire workspace" or manually add the missing pages.

Connector · Required

Connect Google Calendar

⏱ 1 min · OAuth · read-only
+

What Calendar unlocks

Calendar drives Thursday Meeting Prep · the routine that delivers a 3-minute brief on every external attendee in your day.

  • Reads today's external meetings (anything with attendees outside your domain)
  • For each external attendee, scrapes their LinkedIn (via Apify) + recent posts
  • Writes a 3-min brief to Gmail Drafts so it's waiting in your inbox before the meeting

Permissions · read-only

Calendar is the safest connector. Claude only needs READ access. It never creates events, modifies invitations, or sends responses on your behalf.

How to connect (3 clicks)

  1. Connectors page → + → Browse connectors → Google Calendar → Add
  2. Sign in with the same Google account you used for Gmail
  3. Approve "See events from your calendars" + "See attendees". That's all you need.
Calendar shows "● Connected". Test: "What external meetings do I have today?"
Common Calendar issues
"I don't see Thursday Meeting Prep firing." The routine only fires if you have at least 1 external attendee in today's events (someone with an email outside your domain). Internal-only meetings get skipped (you don't need a brief for your own team).

"Multiple Google accounts · which calendar?" Same as Gmail. Connect the calendar of the account where your prospect + client meetings actually live.

Connector · Required

Connect Google Drive

⏱ 1 min · OAuth · MUST allow file creation
+

What Drive unlocks

Drive is where every long-form artifact saves. 3 routines write here:

  • Wednesday Newsletter Draft · saves the full newsletter as a Doc
  • Friday Weekly Wrap · saves the weekly performance summary
  • Sunday Pipeline Report · saves the pipeline status doc

Recommended folder structure

Before you connect, create a Purely Personal folder in your Drive root. Sub-folders inside: Newsletters, Reports, Briefs. Claude saves files into the right sub-folder automatically.

Permissions · CRITICAL

You MUST allow "Create new files." Without this scope, routines can read your Drive but can't write the newsletter draft. The newsletter routine fails silently · you'll wonder why nothing's appearing on Wednesdays.

The 3 permissions Google asks for:

  • ✓ See and download all your Drive files · so Claude can read content for context
  • ✓ Create new files in Drive · CRITICAL · for newsletter + report drafts
  • ✓ Edit your existing Drive files · so updates land in the same draft instead of creating duplicates

How to connect (4 clicks)

  1. Connectors page → + → Browse connectors → Google Drive → Add
  2. Sign in (same Google account)
  3. On the OAuth screen, tick all 3 boxes · do not skip "Create new files"
  4. Approve
Drive shows "● Connected". Test: "Create a test Doc in my Drive called 'PP test' with the body 'hello'." The Doc should appear in your Drive within seconds.
Common Drive issues
"Wednesday newsletter never appears in my Drive." You skipped "Create new files." Disconnect from claude.ai/customize/connectors. Reconnect and tick all 3 boxes.

"I'm hitting Drive storage limits." Old reports pile up. Set a monthly cleanup or use a separate Drive account just for Purely Personal artifacts.

Connector · Optional

Connect Stripe

⏱ 1 min · Skip if you don't use Stripe
+

What Stripe unlocks

Stripe is the only optional connector. It powers /cash-engine · the executive that pulls real revenue numbers automatically.

  • MRR + ARR · current month + annualized recurring revenue
  • Churn · cancellations + downgrades this period
  • 90-day forecast · revenue projection based on current trajectory
  • Sunday Pipeline Report · adds a "Cash collected this week" line

If you skip this connector, /cash-engine still works · it just prompts you to paste your numbers manually each time you run it.

Permissions · read-only

Claude only requests read access to Stripe. Specifically: charges, customers, subscriptions, balance. Claude cannot modify your Stripe account. Cannot issue refunds. Cannot create products. Cannot change anything.

How to connect (3 clicks)

  1. Connectors page → + → Browse connectors → Stripe → Add
  2. Sign in to your Stripe account · pick the right Stripe organization if you have multiple
  3. Approve "Read access to charges, customers, subscriptions, balance"
Stripe shows "● Connected". Test: "Run /cash-engine and pull my MRR." Real numbers should appear within 5 seconds.
Common Stripe issues
"I don't use Stripe · should I skip?" Yes. Skip entirely. /cash-engine works fine without it · it just asks you for numbers each time.

"I use multiple Stripe accounts (multiple businesses)." Pick the one for the business this Brain represents. You can switch later by disconnecting + reconnecting.

"Can Claude move money or issue refunds?" No. Read-only access only. Claude cannot make any change to your Stripe account.

If you're stuck

The 14 most common problems, grouped by phase. Click the one matching what you're seeing. If nothing here helps, scroll to the bottom.

Phase 1 · Install
Windows: PowerShell says 'claude', 'ln', or '$(pwd)' is not recognized
Cause: PowerShell does not understand Unix-style commands. claude may be installed but not on PowerShell's PATH. ln -s, $(pwd), and ~/ paths simply do not exist in PowerShell. Fix: close PowerShell. Open Git Bash from the Start menu (it was installed alongside Git in Step 3). Re-run every command in Git Bash · they all work there. The ONLY command that uses PowerShell is the Step 1 install (iwr ... | iex); after that, switch to Git Bash for everything.
Terminal says command not found: claude
Fix: close and re-open the terminal so PATH refreshes. Mac: run source ~/.zshrc and try again. Windows: open "Environment Variables" in Start menu, check user PATH includes %USERPROFILE%\.claude\bin, re-open Git Bash (not PowerShell).
Terminal says command not found: git
Mac: the xcode-select --install dialog never appeared, or you clicked Cancel. Re-run the command. Windows: the git installer didn't finish or you're not using Git Bash. Open Git Bash from Start menu.
I typed / in Claude Code but no Purely Personal commands appear
Most common cause: you're looking for /build-my-brain but Claude Code 2.x namespaces plugin commands as /purely-personal:build-my-brain. Fix: type /purely-personal · all 9 commands filter into the autocomplete. If they still don't appear: you didn't restart Claude Code. Type /exit (or Ctrl+D), re-run claude, type /purely-personal. If they're still not there: the marketplace wasn't added. Run /plugin marketplace add danielpaulai/Purely-Personal-Run-a-business-by-itself, then /plugin install purely-personal@purely-personal-marketplace, THEN restart.
claude plugin install . says "not recognized" (the OLD instruction)
Why: that command was for an older Claude Code version. Claude Code 2.x has no claude plugin install CLI command at all. Use the new flow: /plugin marketplace add danielpaulai/Purely-Personal-Run-a-business-by-itself from INSIDE Claude Code (not the terminal). Then /plugin install purely-personal@purely-personal-marketplace.
Only 5 or 6 slash commands appear, not 8
Plugin didn't fully install. Inside Claude Code, type /plugin list and confirm purely-personal@purely-personal-marketplace appears. If not, re-run /plugin install purely-personal@purely-personal-marketplace. If still missing, run /plugin marketplace list to confirm the marketplace was added correctly.
Phase 2 · Connectors
I don't see a "+ Add" button on the Connectors page
Claude.ai moved Connectors into Customize. Go to claude.ai/customize/connectors directly. The + button is at the top of the left panel.
Gmail / Notion / Calendar OAuth fails mid-flow
Most common cause: your Google account has "Less secure apps" disabled or you're on a corporate Workspace that blocks third-party OAuth. Ask your IT admin to allowlist Claude, or use a personal Google account for the workshop.
Connector connected but routines fail silently
You didn't grant all permissions during the OAuth flow. Revoke access at myaccount.google.com/permissions, then re-connect and tick all boxes on the consent screen.
Stripe connector won't load
Stripe is optional. Skip Step 11 and move on. /cash-engine will prompt you to paste numbers manually on Day 1.
Phase 3 · Apify MCP
Claude says "I don't have access to Apify" on the test
Three things to check: (1) your JSON is valid. Paste the config into jsonlint.com. Common error: missing comma between servers. (2) Token starts with apify_api_ and is inside quotes. (3) You fully quit Claude Desktop (Cmd+Q on Mac, Task Manager → End Task on Windows), not just closed the window.
Config file doesn't exist at the path
That's fine for a fresh Claude Desktop install. Mac: mkdir -p ~/Library/Application\ Support/Claude && touch ~/Library/Application\ Support/Claude/claude_desktop_config.json. Windows: navigate to %APPDATA%\Claude in File Explorer, right-click → New → Text Document, rename to claude_desktop_config.json (make sure extension isn't .txt).
Apify test command hangs for 30+ seconds
First run is slow because npx downloads the MCP server package. Wait up to 60 seconds. Subsequent calls are instant. If it still hangs: check your internet, then restart Claude Desktop.
Apify returns empty results
Your free tier might be rate-limited if you've run other scrapes today. Check console.apify.com → Billing for your usage. You get $5 free/month. Workshop total use is ~$0.50.

My file saved as BUSINESS-BRAIN.md.txt on Windows
Windows is hiding the .txt extension. In File Explorer: View tab → tick "File name extensions". Rename the file, removing .txt.
/plugins inside Claude Code says "no plugins"
Three possible causes in order. First: commands ARE installed but namespaced. Type /purely-personal instead of /build-my-brain · all 9 appear. Second: you forgot to restart. /exit · re-run claude. Third: marketplace wasn't added. Run /plugin marketplace add danielpaulai/Purely-Personal-Run-a-business-by-itself, then /plugin install purely-personal@purely-personal-marketplace, THEN /exit + re-run claude.
Claude Code won't launch at all
Run claude --version. If that fails, re-install: curl -fsSL https://claude.ai/install.sh | sh (Mac) or iwr https://claude.ai/install.ps1 -UseBasicParsing | iex (Windows). Restart terminal.
