Wire Apify MCP into Claude Desktop

⏱ 6 min · Config file edit
+

MCP = Model Context Protocol. Tells Claude "these external tools are available."

13a · Open the config file

macOS
Windows

In Terminal:

Macopen ~/Library/Application\ Support/Claude/claude_desktop_config.json

In File Explorer address bar:

Windows%APPDATA%\Claude

Open claude_desktop_config.json in Notepad.

13b · Paste this config

First, check if the file is already empty or already has content. Pick the right tab:

Easy case. Paste this whole block into the file. Replace YOUR_TOKEN_HERE with your Apify token. Save.

claude_desktop_config.json · full contents{ "mcpServers": { "apify": { "command": "npx", "args": [ "-y", "@apify/actors-mcp-server", "--actors", "apify/linkedin-profile-scraper,apify/linkedin-company-scraper,apify/google-search-scraper,apify/website-content-crawler,trudax/reddit-scraper-lite" ], "env": { "APIFY_TOKEN": "YOUR_TOKEN_HERE" } } } }

Paste this block as your new "apify" entry (highlighted yellow in the "after" mockup):

Add inside existing mcpServers"apify": { "command": "npx", "args": [ "-y", "@apify/actors-mcp-server", "--actors", "apify/linkedin-profile-scraper,apify/linkedin-company-scraper,apify/google-search-scraper,apify/website-content-crawler,trudax/reddit-scraper-lite" ], "env": { "APIFY_TOKEN": "YOUR_TOKEN_HERE" } }
Two things that break this step: (1) forgetting the comma after the } that closes the previous server, (2) not paying attention to JSON brackets. If Claude Desktop won't load after restart, you have a JSON error. Paste your file into jsonlint.com to find it.

If you're stuck

The 14 most common problems, grouped by phase. Click the one matching what you're seeing. If nothing here helps, scroll to the bottom.

Phase 1 · Install
Terminal says command not found: claude
Fix: close and re-open the terminal so PATH refreshes. Mac: run source ~/.zshrc and try again. Windows: open "Environment Variables" in Start menu, check user PATH includes %USERPROFILE%\.claude\bin, re-open PowerShell.
Terminal says command not found: git
Mac: the xcode-select --install dialog never appeared, or you clicked Cancel. Re-run the command. Windows: the git installer didn't finish or you're not using Git Bash. Open Git Bash from Start menu.
claude plugin install . fails or says "not recognized"
Check 1: you're inside the cloned repo folder. Type pwd (Mac) or cd (Win). The path should end in Purely-Personal-Run-a-business-by-itself. Check 2: your Claude Code is 1.0 or newer. Run claude --version. Fallback: symlink manually with ln -s $(pwd) ~/.claude/plugins/purely-personal.
Only 5 or 6 slash commands appear, not 8
Plugin didn't fully install. Close Claude Code. Run claude plugin list and confirm purely-personal appears. If not, re-run claude plugin install . from inside the repo.
Phase 2 · Connectors
I don't see a "+ Add" button on the Connectors page
Claude.ai moved Connectors into Customize. Go to claude.ai/customize/connectors directly. The + button is at the top of the left panel.
Gmail / Notion / Calendar OAuth fails mid-flow
Most common cause: your Google account has "Less secure apps" disabled or you're on a corporate Workspace that blocks third-party OAuth. Ask your IT admin to allowlist Claude, or use a personal Google account for the workshop.
Connector connected but routines fail silently
You didn't grant all permissions during the OAuth flow. Revoke access at myaccount.google.com/permissions, then re-connect and tick all boxes on the consent screen.
Stripe connector won't load
Stripe is optional. Skip Step 11 and move on. /cash-engine will prompt you to paste numbers manually on Day 1.
Phase 3 · Apify MCP
Claude says "I don't have access to Apify" on the test
Three things to check: (1) your JSON is valid. Paste the config into jsonlint.com. Common error: missing comma between servers. (2) Token starts with apify_api_ and is inside quotes. (3) You fully quit Claude Desktop (Cmd+Q on Mac, Task Manager → End Task on Windows), not just closed the window.
Config file doesn't exist at the path
That's fine for a fresh Claude Desktop install. Mac: mkdir -p ~/Library/Application\ Support/Claude && touch ~/Library/Application\ Support/Claude/claude_desktop_config.json. Windows: navigate to %APPDATA%\Claude in File Explorer, right-click → New → Text Document, rename to claude_desktop_config.json (make sure extension isn't .txt).
Apify test command hangs for 30+ seconds
First run is slow because npx downloads the MCP server package. Wait up to 60 seconds. Subsequent calls are instant. If it still hangs: check your internet, then restart Claude Desktop.
Apify returns empty results
Your free tier might be rate-limited if you've run other scrapes today. Check console.apify.com → Billing for your usage. You get $5 free/month. Workshop total use is ~$0.50.
Phase 4 · Final Prep
I can't find my registration confirmation email
Search your inbox (and Promotions tab) for danny@danielpaul.ai. Not there? Reply to any email from that address with your LinkedIn URL, or DM @danielpaulai on LinkedIn.
My file saved as BUSINESS-BRAIN.md.txt on Windows
Windows is hiding the .txt extension. In File Explorer: View tab → tick "File name extensions". Rename the file, removing .txt.
/plugins inside Claude Code says "no plugins"
You're inside Claude Code but the plugin wasn't installed globally. Go back to the repo folder, run claude plugin install . once more, then cd ~/Desktop/my-business and re-launch claude.
Claude Code won't launch at all
Run claude --version. If that fails, re-install: curl -fsSL https://claude.ai/install.sh | sh (Mac) or iwr https://claude.ai/install.ps1 -UseBasicParsing | iex (Windows). Restart terminal.
