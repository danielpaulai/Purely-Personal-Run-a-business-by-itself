Accounts + Install

Create a Claude account + upgrade to Pro

8 min Everyone

Claude Pro is required for this workshop. You'll also install the Claude Code CLI (the terminal app that runs your engines).

  1. Open claude.ai in your browser
  2. Click Sign up (top right). Use your main email address.
  3. Once logged in, click your profile (top right) → Upgrade to Pro
  4. Pay the $20/month and you're in

Claude.ai homepage Show the "Sign up" button in the top right corner circled in red. screenshots/01a-claude-signup.png
01b-claude-pro-upgrade.png· Pricing page with "Upgrade to Pro" highlighted · $20/mo
Pricing page Show the Pro tier card with "Upgrade" CTA highlighted. Make sure the $20/mo price is visible. screenshots/01b-claude-pro-upgrade.png

Now install Claude Code (the CLI)

Go to claude.com/claude-code and follow the install instructions for your OS.

macOS
Windows

Open Terminal (Applications → Utilities → Terminal) and paste:

Mac Terminalcurl -fsSL https://claude.ai/install.sh | sh

If asked for your password, type it. You won't see characters appear. That's normal.

Open PowerShell (Start → type "PowerShell") and paste:

Windows PowerShelliwr https://claude.ai/install.ps1 -UseBasicParsing | iex

If Windows shows a security warning, click "Run anyway" or re-open PowerShell as Administrator.

01c-claude-code-install-mac.png· Terminal on Mac with install output visible (4 lines of progress)
Terminal with install output Mac: Terminal window showing the install script running, ending with "Installation complete" or similar. screenshots/01c-claude-code-install-mac.png
01d-claude-code-install-win.png· PowerShell on Windows with install output
PowerShell with install output Windows: PowerShell window showing the same successful install. Blue background, white text. screenshots/01d-claude-code-install-win.png
Type claude --version in your terminal. You should see 1.0.x or higher. If you see "command not found", close and re-open your terminal, then try again.
What if claude --version still says "command not found"?
Mac: Run source ~/.zshrc (or source ~/.bash_profile if you're on older macOS). Try claude --version again.

Windows: Fully close PowerShell and re-open it. The PATH needs to refresh. If still not working, search "Environment Variables" in Start menu → check that your user PATH includes %USERPROFILE%\.claude\bin.

Connectors

Connect Gmail

2 min OAuth · no API key

Gmail powers Monday Content Drop, Thursday Meeting Prep, and Friday Weekly Wrap. Claude reads your inbox for audience questions and writes drafts to Gmail Drafts.

  1. Open claude.ai
  2. Click your profile (top right) → Settings
  3. Click Connectors in the left sidebar
  4. Click + Add → pick Gmail
  5. Sign in with Google → allow ALL permissions
Must allow ALL permissions. If you only grant "read" and skip "send drafts", routines will fail silently. Allow everything. Claude only does what the routines tell it to do.

Apify MCP

Create Apify account + API token

5 minFree plan fits workshop

Apify is the one non-OAuth piece. It powers LinkedIn scraping, website crawling, Reddit pain mining, and Google search for the Sales Machine.

Free plan is enough. Apify free tier includes ~$5 of compute per month. The whole workshop uses about 5 compute units. You will not hit the cap.
  1. Open apify.com → Sign up → Free plan → verify email
  2. Log in to console.apify.com
  3. Click Settings (left sidebar) → Integrations
  4. Click + Create new token
  5. Name it: Claude Workshop → click Create
  6. Copy the token immediately. Starts with apify_api_...
Save the token in Notes or your password manager RIGHT NOW. Apify will never show this token again. If you close the page without copying it, you have to create a new one.

13
Apify MCP

Wire Apify MCP into Claude Desktop

6 minConfig file edit

MCP = Model Context Protocol. It tells Claude Desktop "these external tools are available." Once wired, you can say "use Apify to scrape X" in any Claude Code conversation.

Step 13a · Open the config file

macOS
Windows

In Terminal, paste:

Mac Terminalopen ~/Library/Application\ Support/Claude/claude_desktop_config.json

Opens in TextEdit. If the file doesn't exist yet, create it: touch ~/Library/Application\ Support/Claude/claude_desktop_config.json then open it.

In File Explorer address bar, paste:

Windows File Explorer%APPDATA%\Claude

Find claude_desktop_config.json. Open in Notepad. If it doesn't exist, create a new empty file with that exact name.

Step 13b · Paste this config

Replace YOUR_TOKEN_HERE with the Apify token from Step 12. Paste the whole block as-is. If the file already has content, merge the mcpServers section carefully.

claude_desktop_config.json{ "mcpServers": { "apify": { "command": "npx", "args": [ "-y", "@apify/actors-mcp-server", "--actors", "apify/linkedin-profile-scraper,apify/linkedin-company-scraper,apify/google-search-scraper,apify/website-content-crawler,trudax/reddit-scraper-lite" ], "env": { "APIFY_TOKEN": "YOUR_TOKEN_HERE" } } } }
13a-config-file-mac.png· TextEdit on Mac showing the pasted config with token area highlighted
Mac TextEdit with config pasted TextEdit window with the full JSON pasted. Blur the real token. Red-underline the "APIFY_TOKEN" line. screenshots/13a-config-file-mac.png
13b-config-file-win.png· Notepad on Windows showing the same config pasted
Windows Notepad with config pasted Notepad with identical content. Red-underline the APIFY_TOKEN line. File title shows claude_desktop_config.json. screenshots/13b-config-file-win.png

Step 13c · Save + fully quit Claude Desktop

Fully quit, do not just close the window.
Mac: Cmd+Q or click Claude menu → Quit Claude.
Windows: Right-click the Claude icon in system tray → Quit. Or open Task Manager → End Task on Claude.

Now re-open Claude Desktop. The MCP server loads on startup.
